`POST /auth/register`

Creates a new user account. New accounts are assigned the `Viatger` role by default.

<Note>
  Authentication is not required to call this endpoint.
</Note>

## Request

Request body must be JSON.

<ParamField body="email" type="string" required>
  A valid, unique email address. Maximum 50 characters.
</ParamField>

<ParamField body="password" type="string" required>
  Password for the account.
</ParamField>

<ParamField body="full_name" type="string">
  Display name shown in the UI and chat. Maximum 50 characters.
</ParamField>

<ParamField body="rol" type="string" default="Viatger">
  Role to assign. Accepted values: `Viatger`, `Creador`, `Admin`. Defaults to `Viatger`.
</ParamField>

<ParamField body="bio" type="string">
  Short biography. Maximum 200 characters.
</ParamField>

### Example request

```bash
curl -X POST http://localhost:8000/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "maria@example.com",
    "password": "secret123",
    "full_name": "Maria Garcia"
  }'
```

## Response

<ResponseField name="id" type="integer" required>
  Unique identifier for the new user.
</ResponseField>

<ResponseField name="email" type="string" required>
  The registered email address.
</ResponseField>

<ResponseField name="full_name" type="string">
  Display name, or `null` if not provided.
</ResponseField>

<ResponseField name="rol" type="string" required>
  The assigned role. Defaults to `"Viatger"`.
</ResponseField>

<ResponseField name="bio" type="string">
  Biography, or `null` if not provided.
</ResponseField>

### Example response

```json
{
  "id": 42,
  "email": "maria@example.com",
  "full_name": "Maria Garcia",
  "rol": "Viatger",
  "bio": null
}
```

## Errors

| Status | Detail | Cause |
|--------|--------|-------|
| `400` | `Email ya registrado` | An account with that email already exists. |
