`POST /creadorViatges/creator/trips`

Creates a new trip. The authenticated user is recorded as the trip's creator.

<Note>
  Authentication is required. The user must have the `Creador` or `Admin` role.
</Note>

## Request

Request body must be JSON.

<ParamField body="nom" type="string" required>
  Name of the trip. Maximum 50 characters.
</ParamField>

<ParamField body="desti" type="string" required>
  Destination. Maximum 50 characters.
</ParamField>

<ParamField body="data_inici" type="string" required>
  Start date in `YYYY-MM-DD` format.
</ParamField>

<ParamField body="data_fi" type="string" required>
  End date in `YYYY-MM-DD` format.
</ParamField>

<ParamField body="maxim_participants" type="integer" default="10">
  Maximum number of participants. Defaults to `10`.
</ParamField>

<ParamField body="descripcio" type="string">
  Optional description. Maximum 200 characters.
</ParamField>

<ParamField body="estat" type="string" default="Planificant">
  Initial planning status. One of: `Planificant`, `Actiu`, `Complet`, `Cancel·lat`. Defaults to `Planificant`.
</ParamField>

### Example request

```bash
curl -X POST http://localhost:8000/creadorViatges/creator/trips \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "nom": "Ruta pels Pirineus",
    "desti": "Pirineus",
    "data_inici": "2026-07-01",
    "data_fi": "2026-07-10",
    "maxim_participants": 12,
    "descripcio": "Ruta de senderisme pels Pirineus catalans."
  }'
```

## Response

<ResponseField name="id" type="integer" required>
  Unique identifier assigned to the new trip.
</ResponseField>

<ResponseField name="creador_id" type="integer" required>
  ID of the authenticated user who created the trip.
</ResponseField>

<ResponseField name="nom" type="string" required>
  Name of the trip.
</ResponseField>

<ResponseField name="desti" type="string" required>
  Destination of the trip.
</ResponseField>

<ResponseField name="data_inici" type="string" required>
  Start date.
</ResponseField>

<ResponseField name="data_fi" type="string" required>
  End date.
</ResponseField>

<ResponseField name="maxim_participants" type="integer" required>
  Maximum number of participants.
</ResponseField>

<ResponseField name="descripcio" type="string">
  Description, or `null` if not provided.
</ResponseField>

<ResponseField name="estat" type="string" required>
  Planning status.
</ResponseField>

### Example response

```json
{
  "id": 7,
  "creador_id": 5,
  "nom": "Ruta pels Pirineus",
  "desti": "Pirineus",
  "data_inici": "2026-07-01",
  "data_fi": "2026-07-10",
  "maxim_participants": 12,
  "descripcio": "Ruta de senderisme pels Pirineus catalans.",
  "estat": "Planificant"
}
```

## Errors

| Status | Detail | Cause |
|--------|--------|-------|
| `401` | `Token no vàlid` | Missing or invalid JWT token. |
| `403` | `Permiso denegado. Tu rol actual es: <rol>` | Authenticated user does not have the `Creador` or `Admin` role. |
