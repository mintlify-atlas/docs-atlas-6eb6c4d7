---
title: 'List users'
description: 'Retrieve all registered users. Intended for administrators.'
---

`GET /admin/users`

Returns a list of all registered user accounts.

<Note>
  No role check is enforced at the route level, but this endpoint is intended for `Admin` users only.
</Note>

## Request

No parameters.

### Example request

```bash
curl http://localhost:8000/admin/users \
  -H "Authorization: Bearer <token>"
```

## Response

Returns an array of user objects.

<ResponseField name="id" type="integer" required>
  Unique identifier of the user.
</ResponseField>

<ResponseField name="email" type="string" required>
  Email address of the user.
</ResponseField>

<ResponseField name="full_name" type="string">
  Display name, or `null` if not set.
</ResponseField>

<ResponseField name="rol" type="string" required>
  Role of the user: `Viatger`, `Creador`, or `Admin`.
</ResponseField>

<ResponseField name="bio" type="string">
  Biography, or `null` if not set.
</ResponseField>

### Example response

```json
[
  {
    "id": 1,
    "email": "admin@example.com",
    "full_name": "Admin User",
    "rol": "Admin",
    "bio": null
  },
  {
    "id": 42,
    "email": "maria@example.com",
    "full_name": "Maria Garcia",
    "rol": "Viatger",
    "bio": "Apassionada dels viatges i la muntanya."
  }
]
```

## Errors

| Status | Detail | Cause |
|--------|--------|-------|
| `404` | `Usuarios no encontrados` | No users exist in the database. |
