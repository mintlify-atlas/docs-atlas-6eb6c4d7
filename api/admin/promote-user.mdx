---
title: 'Promote user'
description: "Change a user's role."
---

`PUT /admin/users/{user_id}/promote`

Changes the role of the specified user.

<Warning>
  This operation modifies the user's access level. Granting the `Admin` role gives full platform access.
</Warning>

## Request

<ParamField path="user_id" type="integer" required>
  The unique ID of the user whose role to change.
</ParamField>

Request body must be JSON.

<ParamField body="new_rol" type="string" required>
  The new role to assign. One of: `Viatger`, `Creador`, `Admin`.
</ParamField>

### Example request

```bash
curl -X PUT http://localhost:8000/admin/users/42/promote \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"new_rol": "Creador"}'
```

## Response

Returns the updated user object.

<ResponseField name="id" type="integer" required>
  Unique identifier of the user.
</ResponseField>

<ResponseField name="email" type="string" required>
  Email address of the user.
</ResponseField>

<ResponseField name="full_name" type="string">
  Display name, or `null`.
</ResponseField>

<ResponseField name="rol" type="string" required>
  The newly assigned role.
</ResponseField>

<ResponseField name="bio" type="string">
  Biography, or `null`.
</ResponseField>

### Example response

```json
{
  "id": 42,
  "email": "maria@example.com",
  "full_name": "Maria Garcia",
  "rol": "Creador",
  "bio": "Apassionada dels viatges i la muntanya."
}
```

## Errors

| Status | Detail | Cause |
|--------|--------|-------|
| `404` | `Usuario no encontrado` | No user exists with the given `user_id`. |
