Admins are the only users with access to the `/admin/users` and `/admin/requests` routes. The Admin role sits at the top of TravelConnect's three-tier permission system, with full authority over user management, promotion decisions, and trip data.

<Warning>
Some admin actions — such as changing a user's role — take effect immediately and cannot be undone through the UI. Review carefully before confirming any change.
</Warning>

## Admin responsibilities

- **User management** — View every registered user and change their role at any time.
- **Promotion approvals** — Review Creator promotion requests and approve or reject them.
- **Full trip management** — Admins can create and edit any trip, regardless of who created it.

## Accessing admin features

Admins access the administrative sections directly by navigating to these routes in the frontend:

| Route | Purpose |
|---|---|
| `/admin/users` | View all users and change their roles |
| `/admin/requests` | Review and act on pending Creator promotion requests |

These routes are only rendered and accessible to users whose `rol` is `Admin`. Any other role navigating to these paths will not have access.
