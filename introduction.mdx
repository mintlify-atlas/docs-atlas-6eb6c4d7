## What is TravelConnect?

TravelConnect is a full-stack web application that makes group travel social and organized. Users can browse upcoming trips, enroll as participants, chat with fellow travelers, and — if promoted to Creator — plan and publish their own trips.

## Architecture overview

TravelConnect is split into two independent components:

| Layer | Technology | Purpose |
|---|---|---|
| **Backend** | FastAPI (Python) + SQLAlchemy | REST API, auth, business logic, database |
| **Frontend** | React 19 + Vite + React Bootstrap | Single-page application |

The backend exposes a REST API with JWT authentication. The frontend consumes the API via Axios and manages auth state through React Context.

## User roles

TravelConnect has three user roles with escalating privileges:

<CardGroup cols={3}>
  <Card title="Viatger (Traveler)" icon="user">
    Browse trips, enroll / leave, participate in chat, request Creator promotion.
  </Card>
  <Card title="Creador (Creator)" icon="pen-to-square">
    Everything a Traveler can do, plus create and edit their own trips.
  </Card>
  <Card title="Admin" icon="crown">
    Full access: manage all users, approve or reject promotion requests, create and edit any trip.
  </Card>
</CardGroup>

## Key concepts

- **Trip (Viatge)** — A group travel event with a destination, date range, participant cap, and status lifecycle (Planning → Active → Complete / Cancelled).
- **Enrollment** — Any authenticated user can join an available trip up to the participant maximum.
- **Trip Chat** — Participants send and read messages scoped to a single trip.
- **Promotion Request** — Travelers submit a message explaining why they want to become a Creator; Admins approve or reject.
